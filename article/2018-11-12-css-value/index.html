Once a user agent has parsed a document and constructed a document tree, it must assign, for every element in the tree, a value to every property that applies to the target media type.

一旦UA解析完文档并且构造了文档树，它必须为树中每个元素适用于目标媒体类型的属性赋值

本文主要讲解用户代理如果得出最终渲染的属性值的过程，先说一下总的步骤，然后对每一步进行具体的讲解。

注：基本就是CSS2.2文档第6章的内容，加上自己一点点理解，可以直接看文档。

整体步骤
  • 通过指定、继承或者初始值来确定指定值。／the specified value
  • 处理指定值得到一个用于继承的值／computed value
  • 如果需要的话，转化为一个固定的值／the used value
  • 然后根据本地环境限制转化为一个实际应用值。／actual valu
1.确认指定值/ specified value

UA必须按照以下机制（优先顺序）给每一个元素指定值。

  • 如果层叠（cascade）产生了值，就用它。（选取样式表中权重最高的规则）
  • UA（user agent，一般指浏览器）、User（查看页面的人，可以通过浏览器默认样式来修改 default style sheet）、Author（编写页面者）都可以设置元素样式，他们之间会有设置重叠的部分，css cascade 规定了它们的优先级，最终应用优先级最大的。
  • 如果无指定的值而且这个值可继承，使用父元素计算出来的值。
    • root元素没有可以继承的属性值，所以设置root元素没有指定的指之后，直接应用默认值。
  • 否则使用属性的初始值，每个属性的初始值都在属性的定义中。

总结：层叠优先级最高的值 || 继承值 || 初始值

2. 用于继承的计算值／computed value

  • 获取computed value不需要用户代理渲染文档，所以如果指定值对应的值是需要渲染文档才能确认，那么计算值和指定值相同。

    • 如，width: 100% 这个需要等到文档渲染之后，才能确认固定值。而line-height：120%，依据的是font-size的值，可以直接求出来。

  • 如果不依赖渲染文档，计算值取决于属性定义的computed value line决定（在MDN都可以找到），告知不同的属性数据类型对应的计算值计算方法。如果值为inherit，采用与父级属性相同过的指定值。

    • 如：line-height 的 computed value 写的是

    for percentage and length values, the absolute length, otherwise as specified

    • 那么对于百分比，有单位的长度值，直接使用固定的值，百分比乘以font-size 算出固定值，而normal 或者 保持原样。（有单位的值长度值区分 固定（eg: px）和 相对（eg: em）,具体查看文档）
    • 所以可以看到一般推荐使用 来定义line-height，如果使用百分比，对应的line-height 是继承的父元素的固定的值，不由自身font-size 决定。

  • 即使属性不适用当前元素、其计算值也存在（是否适用，在apply to这行）。然而，有些属性可能根据属性是否适用于该元素来定义元素属性的计算值。

  • 注意： 子元素继承父元素的值都对应的父元素的计算值。

3.Used values ／应用值
  • 处理计算值所有依赖（包括依赖的是渲染文档之后才能确认的值），生成固定的值。
  • 比如，width为100%，会计算为固定的值。
4.Actual values／实际值
  • 原则上 应用值可以用于渲染啦，但是现有的浏览器无法直接应用这个值，如PC上写font-size 为 8px。border-radius 为0.5px，比如设定了max-width，所以还需要环境进行特殊处理。
  • 实际值是通过近似处理的应用值
